# SOVA Protocol (Secure Obfuscated Versatile Adapter)

## Обзор
SOVA - это открытый протокол прикладного уровня, предназначенный для конфиденциальной передачи данных в условиях, где провайдеры или промежуточные узлы могут применять глубокий анализ трафика (DPI) для фильтрации или слежки. Основная инновация - динамическая маскировка трафика под легитимные веб-протоколы, что делает невозможным детектирование протокола без блокировки значительной части обычного интернет-трафика.

## Ключевые особенности
- **Открытый исходный код**: Код опубликован под лицензией MIT/GPLv3
- **Простота установки**: Развертывание сервера одной командой `curl | bash`, клиент одной командой в PowerShell или bash
- **Конфигурация**: JSON-ссылка, содержащая все параметры подключения (публичный ключ сервера, список транспортов, предпочтительные SNI для маскировки)
- **Кроссплатформенность**: Сервер на Linux (amd64, arm64), клиенты на Windows, macOS, Linux, Android, iOS
- **Автономность**: Протокол сам адаптируется к изменениям сети без участия пользователя и без обращения к внешним AI-сервисам
- **AI-умная адаптация**: Встроенный интеллект для обнаружения DPI и автоматического переключения тактик
- **Непробиваемая безопасность**: Лучшее шифрование, ZKP-аутентификация, пост-квантовое шифрование

## Архитектура
### Идентификация и аутентификация
Используется пара user_id:password вместо UUID. Пароль доказывается через Zero-Knowledge Proof (на основе ed25519 и пост-квантовых алгоритмов) при handshake, сервер не хранит пароли в открытом виде.

Каждый сервер генерирует свою ключевую пару, публичный ключ включается в JSON-ссылку.

### ⚡ Инновационные компоненты v1.0.0+

#### 1. **Автоматическое обнаружение каналов связи (ConnectivityDetector)**
- 🌐 Проверка интернета (многотаргетная)
- 📶 Сканирование Wi-Fi сетей
- 🛰️ **Обнаружение сотовых вышек** (5G, 4G, 3G, 2G)
- 📡 Поиск соседних mesh-узлов через мультикаст
- 🚀 Автоматический failo ver между каналами

#### 2. **Mesh-сетевая архитектура (MeshNetwork)**
- **Peer-to-Peer туннелирование**: когда интернет недоступен, узлы автоматически создают mesh-сеть
- **Многохопная маршрутизация**: данные передаются через соседние узлы (как Tor)
- **Автоматический relay**: каждый узел может использоваться как ретранслятор
- **Шифрованный gossip**: все message обзываются и шифруются
- **Таблица маршрутов**: динамически обновляется при изменении сети

#### 3. **Offline-First архитектура (OfflineFirstArchitecture)**
Когда интернет полностью отключен, система полностью автономна:

- ✅ **Локальное кэширование**: контент и конфигурация кэшируются на устройстве
- ✅ **Обнаружение близких устройств**: поиск Bluetooth, NFC, Short-Range Radio
- ✅ **Адаптивная маршрутизация**: выбирает оптимальный маршрут из доступных
- ✅ **Управление ресурсами**: мониторинг батареи, CPU, памяти, места
- ✅ **Режим критической нехватки**: оптимизация при < 5% батареи
- ✅ **Рассчет выживаемости**: предсказание как долго система может работать офлайн

#### 4. **AI-Powered Routing (AdaptiveRouter)**
- 🤖 Анализ сетевых аномалий в реальном времени
- 🤖 Автоматическое переключение маршрутов при сбое
- 🤖 Предсказание задержек и выбор оптимального канала
- 🤖 Самообучающаяся система для обхода DPI

#### 5. **Автоматический поиск выхода в интернет**
При полной блокировке интернета система:
1. Сканирует доступные сотовые вышки 
2. Пытается подключиться через мобильного оператора
3. Ищет открытые Wi-Fi сети
4. Создает mesh-сеть с другими устройствами SOVA
5. Использует Bluetooth для связи с близкими узлами
6. Кэширует и повторно использует ранее полученные данные

### Транспортные режимы (уровни маскировки)
Протокол поддерживает три режима, выбираемых автоматически клиентом на основе успешности соединения и инструкций от сервера:

1. **Web Mirror Mode** (TCP-порт 443) - Маскировка под HTTPS с custom handshake
2. **Cloud Carrier Mode** (UDP/QUIC) - QUIC с jitter и адаптивным congestion control
3. **Shadow WebSocket Mode** - WebSocket через CDN с ротацией IP

### Интеллектуальная адаптация
Клиент постоянно измеряет RTT, отслеживает потерю пакетов и появление RST-пакетов или HTTP-заглушек.

При обнаружении аномалий (признак активного DPI) AI-адаптер:
- Переключается на другой транспортный режим
- Меняет паттерн фрагментации
- Меняет целевой SNI из заранее загруженного списка
- Применяет timing jitter и packet morphing
- **Ищет альтернативные каналы** (вышки, mesh-узлы, мобильные сети)

## Установка
`SOVA` распространяется в виде **готовых статических бинарников** для всех поддерживаемых ОС; никаких исходников или компилятора устанавливать не требуется. Исходный код остаётся открытым для разработчиков, но на клиентских и серверных системах он не нужен. Язык Go и любые другие среды отсутствуют на целевых машинах — вы получаете полностью автономную программу.

### Команда установки (любая платформа)
```bash
# единственная команда устанавливает сервер или клиент в зависимости от системы
curl -sSL https://github.com/IvanChernykh/SOVA/releases/download/v1.0.0/install.sh | bash   # на Linux/Unix
powershell -NoProfile -Command "iex ((New-Object Net.WebClient).DownloadString('https://github.com/IvanChernykh/SOVA/releases/download/v1.0.0/install.ps1'))"  # Windows
```
Эти скрипты автоматически определяют архитектуру и загружают соответствующий бинарник из [GitHub Releases](https://github.com/IvanChernykh/SOVA/releases/latest), устанавливают `sova` в систему и настраивают сервис/демон.

После установки доступна команда `sova` с подкомандами (`install`, `connect`, `disconnect`, `status`, `config`, `proxy` и др.). Команда `sova install` выполняет первоначальную настройку (создание сервисов/демонов, генерацию ключей, выдачу JSON-ссылки).

## Этапы реализации
- Этап 0: Исследование и прототип (готов)
- Этап 1: Базовый транспорт с custom handshake
- Этап 2: QUIC-режим с пост-квантовым шифрованием
- Этап 3: WebSocket через CDN
- Этап 4: Полная AI-адаптация
- Этап 5: CLI с UI и упаковка

## Безопасность
- **Шифрование**: AES-256-GCM + ChaCha20-Poly1305 + пост-квантовые алгоритмы (Kyber, Dilithium)
- **Аутентификация**: ZKP на ed25519 + Schnorr signatures
- **Обфускация**: Dynamic packet morphing, timing jitter, SNI rotation
- **Защита от DPI**: Custom TLS fingerprints, QUIC masquerading, CDN tunneling

## Совместимость и API
SOVA ядро может интегрироваться как плагин в:
- **Xray/V2Ray**: Расширение протокола VLESS+XTLS
- **Sing-Box**: Нативная поддержка outbound
- **Другие клиенты**: Через локальный SOCKS5 прокси
Официальный репозиторий доступен по адресу: https://github.com/IvanChernykh/SOVA . Релизы публикуются в разделе **[Releases](https://github.com/IvanChernykh/SOVA/releases)**, а команда `sova install` всегда подтягивает последнюю версию.
Сервер предоставляет открытый REST API по `/api/*`:
- `GET /api/status` – статистика и uptime
- `POST /api/register` – регистрация пользователя
- `GET /api/config?user_id=<id>` – выдача base64 JSON-конфига
- `GET /api/export?client=<xray|singbox>&user_id=<id>` – вывести конфиг для клиента
- `GET /api/proxy` – готовые ссылки для популярных прокси-клиентов

Эти возможности позволяют любым приложениям и языкам автоматически настраиваться на использование SOVA.
